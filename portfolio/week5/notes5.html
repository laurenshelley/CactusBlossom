WEEK 5 READING/NOTES
Stack traces
-A slack trace is when an error is detected. It works by starting where
the problem occurred so you can find the source of the problem.
Warnings
-Warnings are nice because they will crash the application but it will not
fix itself. It will simply state that there is a problem and
the application will keep running. So if it’s a bigger issue, the issue will persist till the warning is fixed.
Purpose and importance of debugging
-Debugging is important for the user’s sake. No one wants to use an app that
is always crashing or isn’t working properly. Its important to
always check your code because errors will not always appear when using javascript.
Strict Mode
-This is used to detect any errors or potential errors early on in coding. While
this my be annoying to always get errors it is better in the
long run because when you finish you know your code is working perfectly.
Linting Tools
-This can be used for consistency. It’s great for group work because it will make
sure everyone is following the same guideline avoiding messy
work that doesn’t match down the road.
Feature Detection
-This I think is really clever. Since not all code will be supported by all browsers
you can use an if statement in case code isn’t supported. By using an If statement you can say that IF this browser doesn’t support A use B.
Debugging in Browser
-A good method of debugging is taking the code one section at a time. That way you
can rule out the good code and single out the code that isn’t working. An easy way to do this is by inspecting the code on a browser.
Using the Console
-sing console.log is an amazing way to check if your code is working. It’s nice
because you can put it almost anywhere on your code and it will tell
you exactly what isn’t working through different errors. It can also tell you which line of code is where the problem began.
